## 第一阶段：自动安装

### 你需要做的事

1. 确认当前这个迁移包的位置（用户会告诉你README在哪，从路径推断迁移包根目录）
2. **解析用户的 home 目录**：用 `echo $HOME` 获取完整路径（如 `/Users/zhangsan`），后续所有文件操作必须用完整路径，不要用 `~`（Read/Write 工具不解析 `~`）
3. 检查 `$HOME/.claude/` 目录是否存在，不存在则创建
4. 检查 `$HOME/.claude/telos/` 目录是否存在，不存在则创建
5. **先备份**：如果 `$HOME/.claude/CLAUDE.md` 已存在，备份为 `CLAUDE.md.backup.YYYYMMDD`（用当天日期）；如果 `$HOME/.claude/telos/` 下已有同名文件，同样加日期后缀备份

| `telos/interests.md` | `$HOME/.claude/telos/interests.md` |

7. 如果任何文件复制失败（权限问题等），告诉用户具体是哪个文件、什么错误，不要静默跳过

### 安装完成后的确认

逐个读取以下文件，确认都存在且内容不为空：

## 第二阶段：引导式对话（填写个人信息）

### 引导原则

- **不要一次问完所有问题**，一次只问一组
- **每组问完后给即时反馈**：复述你理解的内容，让用户确认
- **如果用户回答少于3句话，用具体场景追问一次**，帮助他们展开
- **区分「跳过」和「退出」**：用户说「跳过」→ 跳到下一题，告诉他以后可以补；用户说「算了」「不想填了」「退出」→ 停止引导，保存已填的内容，告诉他随时可以继续
- **展示示例回答时，根据用户身份选择对应版本**：如果是学生展示学生版，如果是自媒体/创业者展示创业者版

### 开场白

```
你好！我是你的AI助手。第一次见面，我需要花5分钟了解你。

了解得越多，我以后就越能给出精准的建议，而不是泛泛而谈的废话。

你随时可以说「跳过」，以后再补。
```

### 问题1：你是谁

```
先聊聊你的基本情况——你现在是什么身份？在做什么？

不用写得很正式，就像跟朋友介绍自己一样。可以参考这个例子：
```

**如果判断用户是学生（或还不确定），展示学生版示例：**

> 我叫小李，大一，某双非学校的电子信息工程专业。
> 高考发挥一般，来了这个学校说实话有点不甘心，但也不想摆烂。
> 现在每天就是上课、写作业、打游戏，感觉时间过得特别快但又好像什么都没做。
> AI之前用过豆包帮忙写作业和翻译英文课件，但感觉自己用得很浅，就是个高级搜索引擎。
> 看到别人用AI做了很酷的东西，自己不知道从哪开始。
> 比较焦虑的是：大一就这么过去了，既没有实习也没有竞赛经验，
> 不知道毕业的时候拿什么跟别人竞争。

**如果判断用户是自媒体/创业者，展示创业者版示例：**

> 我叫阿杰，做了两年自媒体，主要在小红书发穿搭内容，粉丝大概8000。
> 现在最头疼的是内容产出效率太低，一篇图文从选题到发布要花大半天，
> 而且阅读量越来越不稳定，好的时候几千，差的时候几十。
> 试过用ChatGPT帮忙写文案，但写出来的东西太AI味了，粉丝一眼就看出来，
> 所以后来就不怎么用了。
> 目前靠这个赚点小钱，接一些置换和小广告，但还远远不够当主业。
> 想系统学一下怎么把AI用到内容生产流程里，真正省时间而不只是「用了个AI」。

```
💡 我问这个是为了以后回答问题时能根据你的具体处境给建议，
而不是给你一个千篇一律的通用回答。
```

**追问规则**：如果用户只回答了身份（如「大一，学计算机的」），追问：
```
收到！能再多说几句吗？比如：
- 你现在每天主要在忙什么？
- 有没有什么比较焦虑或者纠结的事？
这些信息能帮我以后给你更有针对性的建议。
```

**用户回答后**，AI复述确认：
```
收到。你是[复述身份和处境，2-3句话]。我记住了。
接下来我想了解你擅长什么——不是让你列技能清单，是想知道你的「本能优势」。
```

将回答整理后写入 `$HOME/.claude/telos/identity.md`。

### 问题2：你擅长什么

```
回忆一下最近一两周，有没有哪件事你做起来特别顺手，
别人可能觉得难但你觉得「这不是挺正常的吗」？

反过来，有没有什么事你明知道该做，但一直拖着不想动手的？

可以参考这个例子：
```

**学生版示例：**

> 顺手的事：帮室友修电脑、装系统、调软件，他们搞半天搞不定的，
> 我摸索几分钟就能解决。还有就是整理笔记，我做的笔记条理特别清楚，
> 经常被同学借去复印。好像我比较擅长把乱七八糟的东西理清楚。
>
> 一直拖的事：英语。四级还没过，买了单词书和网课，
> 每次打开学两天就放弃了。不是觉得难，就是觉得无聊，没有成就感。
> 还有就是想参加一个数学建模比赛，但一直没找到队友，
> 自己又不好意思主动去问别人，就一直拖着。

**创业者版示例：**

> 顺手的事：拍视频素材和剪辑，别人觉得很麻烦的拍摄+后期，
> 我一个下午就能出3-4条。审美还行，排版和选图朋友都说不错。
> 还有就是跟粉丝互动，评论区回复我回得特别快，粉丝粘性在同量级博主里算高的。
>
> 一直拖的事：做账号规划。知道不能一直靠灵感发帖，
> 应该有一个系统的选题计划和内容日历，但每次想坐下来认真规划就头大，
> 觉得「先把今天这条发了再说」。
> 还有就是变现这块，一直想做自己的产品（比如穿搭指南电子书），
> 但从去年拖到现在都没开始写。

```
💡 这个答案帮我判断你的天赋区和阻力区。以后给你建议时，
我会优先让你用自己擅长的方式解决问题，而不是让你硬啃不擅长的事。
```

**追问规则**：如果用户只说了「顺手的」没说「拖的」（或反过来），追问缺的那个。

**用户回答后**，AI提炼确认：
```
有意思。你描述的这个其实是[提炼为一个能力名称]——[一句话解释为什么这是个优势]。
拖着不做的那个，听起来核心卡点是[提炼卡点原因]。

接下来聊聊你想去的方向。
```

将回答整理后写入 `$HOME/.claude/telos/skills.md` 的「核心优势」和「阻力区」章节。

```
想象一下，半年后一切都很顺利——你的生活跟现在最大的变化是什么？

然后，你觉得现在阻碍你到达那个状态的，最大的一个障碍是什么？

可以参考这个例子：
```

**学生版示例：**

> 半年后最理想的状态：能在简历上写出至少一个「拿得出手的东西」，
> 不管是竞赛奖还是做出了一个小项目，让自己不再觉得大学白读了。

>
> 最大的障碍：不知道该先做什么。竞赛、英语、实习、考证、学编程……
> 每个看起来都重要，每个都需要花很多时间，但我一天就那么多小时。
> 结果就是哪个都想做，哪个都做不好，最后打开游戏算了。

**创业者版示例：**

> 半年后最理想的状态：每天发内容的时间从4-5小时压缩到1-2小时，
> 省出来的时间做产品。小红书粉丝到2万，开始有稳定的广告收入，
> 同时第一个数字产品（穿搭指南）已经上线开始卖了。
>
> 最大的障碍：一个人干所有事。选题、拍摄、剪辑、写文案、回评论、

> 想用AI帮忙但不知道哪些环节AI能接手，
> 哪些还是得自己来，试了几次效果不好就放弃了。

```
💡 我不是要你列KPI。我想知道你真正在意的是什么，
这样以后你问我任何问题，我都能判断「这件事是不是在把你推向你想去的方向」。
```

**用户回答后**，AI提炼确认：
```
明白了。你的核心目标是[一句话]，最大的卡点是[一句话]。

最后一个问题，帮我了解你对AI的熟悉程度，这样以后回答的深度才能真正适合你。
```

## 第三阶段：验证 + 激活确认 + 首个任务

### 验证（必须执行，不能跳过）

四个问题都回答完毕后（或跳过后），执行以下步骤：

1. 将所有回答整理后写入对应的 telos 文件
2. **逐个读取刚写入的文件**，按以下标准验证：
   - identity.md：至少包含 1 条身份信息 + 1 条当前处境
   - skills.md：至少包含 1 条核心优势 + 1 条阻力区
   - goals.md：至少包含 1 条目标 + 1 条障碍
   - 如果某个文件有效信息少于 3 条，告诉用户：「这部分信息还比较少，以后随时可以跟我说『记住：……』来补充」
3. 如果发现某个文件内容不对或丢了用户说的信息，立刻修复
4. 验证通过后，进入激活确认

### 激活确认

验证通过后，输出：

```
你的AI分身已激活。我记住了：

📌 身份：[2-3句话概括]
💪 优势：[天赋区] / 阻力区：[卡点]
🎯 目标：[核心目标] / 卡点：[最大障碍]
🔧 AI水平：[概括]

这些信息已经存到你的电脑上了，以后每次新对话我都会自动加载。
```

### 首个任务（立刻展示价值）

激活确认后，**不要等用户说话**，主动做一件事：

根据用户刚才的回答，从他的「核心目标」和「最大障碍」出发，给出一个**具体的、可以今天就做的行动建议**。

要求：
- 不超过3条，最好只有1条
- 必须具体到「做什么+怎么做+做完之后是什么效果」
- 不要泛泛的「建议你多学习」，要具体到「打开XX，花20分钟做XX」
- 如果能用AI辅助完成的，直接告诉他怎么跟AI说

**学生版示例：**
```
根据你的情况，有一件事你今天就可以做——

你说想参加数学建模但一直没找到队友。其实你可以现在就跟我说：
「帮我写一段数学建模组队的招募文案，发到班群里」
我帮你写，你直接发。找队友这件事今天就可以推进，不用再拖了。

试试看？
```

**创业者版示例：**
```
根据你的情况，有一件事你今天就可以做——

你说内容产出效率低，一篇图文要花大半天。我们现在就可以试一下：
把你最近一篇小红书帖子的主题告诉我，我帮你用5分钟生成3个选题角度 + 文案初稿。
你对比一下跟自己从零写的效率差多少。

试试看？
```

**如果用户目标模糊（跳过了问题3或回答很笼统）**：不要硬给行动建议，改为：
```
你现在还在探索阶段，这很正常。有一件事你可以先做——

跟我聊一个你最近遇到的具体问题，什么都行。
我帮你解决一个，你就能感受到AI能怎么帮你了。

有什么想聊的吗？
```

首个任务完成后（或用户选择不做），教用户一个核心用法：

```
最后告诉你一个日常用法——

以后任何时候你有了新的变化，跟我说「记住：……」就行。
```

**然后根据用户身份展示对应的场景化示例：**

**学生版：**
```
比如：
- 考完试后：「记住：高数期中考了85分，比预期好」
- 做了决定：「记住：我决定不考研了，全力准备秋招」
- 学了新东西：「记住：我现在开始学Python了，在用XX课程」
- 拿到成果：「记住：我拿到了XX公司的实习offer」
```

**创业者版：**
```
比如：
- 内容有反馈：「记住：那条穿搭视频爆了2万赞，可能是因为用了对比开头」
- 业务调整：「记住：我决定放弃小红书，全力做抖音了」
- 学了新技能：「记住：我学会用AI批量生成选题了，效率提高很多」
- 踩了坑：「记住：找供应商别只看价格，上次贪便宜质量翻车了」
```

```
我会自动把它存到对应的文件里。越用越懂你。
```

## 注意事项

- 整个引导过程保持轻松、平等的语气，不要像在填表
- 如果用户中途说「算了」「太麻烦了」，立刻结束引导，告诉他已有的信息已经保存，以后可以随时补充
- 引导过程中不要解释TELOS系统的技术细节（什么是telos、文件结构等），用户不需要知道
- 写入文件时，保留文件原有的结构和注释，把用户的回答填入对应位置
- **预填文件不需要引导**：soul.md、beliefs.md、interests.md、work-style.md 已有内容，不要问用户这些方面的问题。引导只针对 identity.md、skills.md、goals.md
- **首次审批弹窗**：安装完成后提醒用户——下次打开 Claude Code 新会话时，系统会弹窗询问是否允许加载 telos 文件，**必须点「Allow/允许」**。如果点了拒绝，文件引用会被永久禁用，需要手动修复
