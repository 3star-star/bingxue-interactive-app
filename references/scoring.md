# 表达力评分维度与标准

## 四个评分维度

### 1. 场景清晰度（Context Clarity）

你是谁、在什么情况下、为什么需要这个。


| 评分  | 标准          | 典型表现                                         |
| --- | ----------- | -------------------------------------------- |
| ⭐☆☆ | 没有任何背景信息    | 「帮我写个方案」——谁的方案？什么行业？给谁看？全靠猜                  |
| ⭐⭐☆ | 有部分背景但不完整   | 「我是做教育的，帮我写个招生方案」——知道行业了，但不知道什么教育、什么阶段、招什么学生 |
| ⭐⭐⭐ | 背景完整，AI不需要猜 | 「我是一个面向大学生的AI课程讲师，8节直播课，已上4节，现在要写第5节的招生复购方案」 |


### 2. 目标精确度（Objective Precision）

你要做什么，具体到什么程度。


| 评分  | 标准           | 典型表现                                              |
| --- | ------------ | ------------------------------------------------- |
| ⭐☆☆ | 目标模糊或使用抽象词   | 「帮我做得更好」「写个高质量的文案」「优化一下」——好是什么？高质量是什么？            |
| ⭐⭐☆ | 目标明确但缺乏具体指标  | 「帮我写一篇关于AI教育的文章」——知道写什么了，但多长？什么风格？发在哪？            |
| ⭐⭐⭐ | 目标具体、可衡量、有边界 | 「帮我写一篇2000字的公众号文章，主题是AI如何帮大学生提高学习效率，面向完全不懂AI的大学生」 |


### 3. 约束完整度（Constraint Coverage）

不要什么、范围多大、什么条件下。


| 评分  | 标准         | 典型表现                                  |
| --- | ---------- | ------------------------------------- |
| ⭐☆☆ | 没有任何约束或限制  | 完全开放式，AI只能按默认方式做，大概率不是你想要的            |
| ⭐⭐☆ | 有部分约束但不完整  | 「不要太长」——多长算太长？「风格轻松一点」——轻松到什么程度？      |
| ⭐⭐⭐ | 约束明确，划清了边界 | 「800字以内，口语化，不要用专业术语，面向零基础读者，不需要讲技术原理」 |


### 4. 可执行度（Actionability）

AI 拿到这段话能不能直接动手，不需要再猜。


| 评分  | 标准              | 典型表现                         |
| --- | --------------- | ---------------------------- |
| ⭐☆☆ | AI 必须猜测多个关键信息   | 缺少受众、格式、用途等关键信息，AI只能全猜       |
| ⭐⭐☆ | AI 需要猜测 1-2 个信息 | 大部分清楚，但有一两个关键维度没说（如格式或用途）    |
| ⭐⭐⭐ | AI 可以直接执行，零猜测   | 所有关键信息完备，AI拿到就能开始做，产出大概率一次到位 |


## 总分计算

每维度 1-3 星，总分 4-12 星：


| 总分      | 等级     | 建议             |
| ------- | ------ | -------------- |
| 4-6 星   | 需要大量澄清 | 用户需要练习把需求说清楚   |
| 7-9 星   | 基本清楚   | 还有改进空间，但已经可以工作 |
| 10-12 星 | 表达很清晰  | 可以直接执行，表达力很强   |


## 评分后的反馈模板

评分后给一句**具体的、可操作的**改进建议，不要泛泛而谈。

好的反馈：

> 「你的目标很清楚，但缺少约束——你没说这个方案要多长、给谁看、什么场景用。下次试试加一句：做到____程度就够了。」

差的反馈（避免）：

> 「建议你下次描述得更清楚一些。」——这本身就是一个模糊的建议

## 更多评分案例（按维度）

### 场景清晰度 — 更多案例

| 用户说的 | 评分 | 为什么 |
|---------|------|--------|
| 「帮我做个方案」 | ⭐☆☆ | 谁做？什么行业？给谁？全靠猜 |
| 「帮我写条朋友圈」 | ⭐☆☆ | 什么场景？卖东西？晒生活？表态度？ |
| 「我是做AI课的，帮我写招生文案」 | ⭐⭐☆ | 知道行业了，但不知道课卖给谁、什么价位、什么阶段 |
| 「我刚做了一个小程序，想在朋友圈发个推广」 | ⭐⭐☆ | 知道了具体事件，但不知道小程序做什么、面向谁 |
| 「我是AI课讲师，150人大学生社群，课上到第4节，想设计一个打卡活动提升完课率」 | ⭐⭐⭐ | 身份+规模+进度+目的，全清楚 |

### 目标精确度 — 更多案例

| 用户说的 | 评分 | 为什么 |
|---------|------|--------|
| 「帮我做得更好」 | ⭐☆☆ | 什么东西做更好？好的标准是什么？ |
| 「优化一下」 | ⭐☆☆ | 优化什么？往哪个方向优化？ |
| 「帮我写一篇介绍AI的文章」 | ⭐⭐☆ | 知道写什么了，但多长？发哪？给谁看？ |
| 「帮我写一个7天打卡活动方案」 | ⭐⭐☆ | 知道做什么了，但打卡什么内容？什么算成功？ |
| 「帮我写一篇2000字公众号文章，讲AI如何帮大学生写论文，面向零基础大学生」 | ⭐⭐⭐ | 字数+平台+主题+受众，都清楚 |

### 约束完整度 — 更多案例

| 用户说的 | 评分 | 为什么 |
|---------|------|--------|
| （完全没提任何限制） | ⭐☆☆ | AI 只能按最通用的方式来 |
| 「不要太长」 | ⭐⭐☆ | 有约束但不可执行——多长算太长？ |
| 「轻松一点的风格」 | ⭐⭐☆ | 有方向但不精确——轻松到什么程度？ |
| 「口语化、800字以内、不用专业术语」 | ⭐⭐⭐ | 多个可执行的约束，AI 可以据此判断 |
| 「不要PPT式的条条框框，要像朋友聊天一样，类似李诞那种风格」 | ⭐⭐⭐ | 排除法+正面参考，AI 精准定位 |

### 可执行度 — 更多案例

| 用户说的 | 评分 | 为什么 |
|---------|------|--------|
| 「帮我搞一下」 | ⭐☆☆ | 搞什么？怎么搞？搞到什么程度？全猜 |
| 「帮我分析这个数据」（附了文件但没说分析什么） | ⭐⭐☆ | 有数据了，但不知道要分析什么维度、给谁看 |
| 「帮我做个小程序」（没说做什么功能） | ⭐☆☆ | AI 连第一行代码都不知道写什么 |
| 「帮我做一个论文聚合搜索小程序，聚合知网+万方+维普，MVP只做搜索功能」 | ⭐⭐⭐ | 功能+数据源+范围全部明确 |
